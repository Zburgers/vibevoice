# VibeVoice 0.2.6

VibeVoice 0.2.6 is a reliability and usability update that makes everyday dictation safer, clearer, and easier to control.

## What’s new for users

### Safer transcript history

Saved transcripts are now much better protected against accidental loss or corruption. VibeVoice writes history more safely, keeps a recovery backup, and can automatically recover when the main history file is damaged.

Your history remains local and is still only saved when you enable it.

### Recording timers now update correctly

The elapsed recording time now moves every second in both the main dashboard and the floating pill, so you can immediately see how long you have been recording.

### The floating pill stays on screen

Expanding or collapsing the pill near the edge of a monitor no longer allows it to spill outside the visible work area. This also works better across scaled and multi-monitor setups.

### Easier window resizing

The main VibeVoice window can once again be resized normally from its edges and corners.

### More control over the floating pill

A new **Keep pill above other windows** setting lets you choose whether the pill should remain always visible. It is enabled by default, so existing users keep the current behavior unless they turn it off.

### Better pill positioning

The collapsed pill and its drag handle now stay aligned, making the control easier to move without an awkward gap.

## Privacy and compatibility

- Speech transcription remains local through `whisper.cpp`.
- Transcript history remains opt-in.
- Existing settings continue to work after updating.
- The new always-on-top option defaults to the previous behavior.

## Technical details

### History reliability

- Serializes every history read-modify-write path, including append, delete, clear, export cleanup, and retention cleanup.
- Replaces direct JSON writes with same-directory atomic replacement and flushes completed writes before replacement.
- Maintains a valid `history.json.bak` recovery copy.
- Preserves malformed history as `history.corrupt-*.json` before restoring the last valid backup or creating a valid empty history.
- Adds regression coverage for concurrent mutations and corrupt-file recovery.

### Desktop fixes

- Recomputes elapsed recording time on every timer tick in the dashboard and pill.
- Clamps expanded and collapsed pill geometry to the active monitor work area.
- Aligns the initial collapsed window dimensions with the actual pill layout.
- Adds resize hit targets to the undecorated main window’s edges and corners.

### Release maintenance

- Updates all application version declarations to `0.2.6`.
- Tracks the resolved `atomicwrites` dependency in `Cargo.lock` for reproducible builds.
- Repairs and validates the Windows, Linux, and macOS release workflow.
