# VibeVoice 0.2.0 Release Notes

Release date: 2026-07-02

## Summary

VibeVoice 0.2.0 is a responsiveness and reliability release. The stop-recording path no longer performs transcription, clipboard writes, paste simulation, and history persistence inside the foreground command. Instead, stopping a recording immediately transitions the app into `Processing`, emits a state-change event, and finishes the expensive work on a background worker.

## What Changed

- Added background transcription processing after recording stop to prevent the UI from freezing while `whisper-cli` runs.
- Replaced continuous frontend polling with backend-driven `vibevoice-state-changed` events.
- Kept a short refresh loop only while actively recording so elapsed time and mic-level display still update.
- Adopted Tauri's cross-platform clipboard manager for clipboard writes instead of shelling out to `wl-copy`, `xclip`, or PowerShell for copy.
- Moved explicit paste execution onto an async blocking worker for manual re-insert commands.
- Standardized output status handling so paste failures keep successful copied text available and copy failures are reported consistently.
- Added Rust unit tests for clipboard capability reporting and copy/paste status/error mapping.
- Replaced the native main-window titlebar with an integrated app titlebar and in-app minimize, maximize, and close controls.
- Reworked the floating pill into a small circular collapsed control with a dedicated drag grip.
- Simplified expanded pill mode to only show useful quick actions: move, collapse, record/stop, paste previous, open app, and the latest transcript/error.
- Updated app metadata to version `0.2.0`.

## User Impact

- Stopping a recording should feel immediate, even when local transcription takes longer.
- The pill and main window now update when the backend state changes instead of waiting on a one-second global poll.
- The main window no longer shows the extra native titlebar above the app UI.
- The collapsed pill takes up far less screen space and is easier to move.
- If auto-paste fails, the transcript remains copied and the history entry records the paste error.
- If clipboard copy fails, the app records `clipboard_failed` and surfaces the clipboard error clearly.

## Compatibility Notes

- Whisper engine discovery and the existing `auto` path behavior are unchanged.
- Linux recording still uses the runtime-native recorder fallback order: `pw-record`, then `arecord`, then `ffmpeg`.
- Paste simulation still depends on native desktop helpers: PowerShell SendKeys on Windows, or `wtype`, `xdotool`, or `ydotool` on Linux.
- The app now requires the Tauri clipboard manager permission `clipboard-manager:allow-write-text`.
